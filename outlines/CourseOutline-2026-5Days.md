## ภาพรวมหลักสูตร

ปี 2026 ถือเป็นจุดเปลี่ยนสำคัญของวงการพัฒนาซอฟต์แวร์ จากยุค "AI ช่วยเติมโค้ด (Autocomplete)" สู่ยุค **Agentic Software Engineering** ที่ AI สามารถวางแผน เขียน แก้ไข ทดสอบ และส่งมอบฟีเจอร์ได้แบบกึ่งอัตโนมัติภายใต้การกำกับของนักพัฒนา นักพัฒนายุคใหม่จึงไม่ได้แข่งกันที่ "พิมพ์โค้ดเร็วแค่ไหน" แต่แข่งกันที่ความสามารถในการ **สื่อสารเจตนา (Intent)**, **ออกแบบบริบท (Context Engineering)** และ **กำกับ AI Agent** ให้ทำงานได้อย่างถูกต้อง ปลอดภัย และมีคุณภาพ

หลักสูตร **"Generative AI for Software Developer"** ฉบับ 5 วันนี้ ขยายต่อยอดจากฉบับ 3 วัน โดยเพิ่ม **2 วันเชิงลึกด้าน AI Engineering และการนำระบบ AI ขึ้น Production (LLMOps)** เหมาะสำหรับผู้ที่ต้องการก้าวจาก "ผู้ใช้ AI ช่วยพัฒนา" ไปสู่ "ผู้สร้างแอปพลิเคชันที่ขับเคลื่อนด้วย AI" ได้จริง ครอบคลุมตั้งแต่พื้นฐาน LLM, เทคนิค Prompt/Context Engineering, การใช้ AI Coding Agent, การเชื่อมต่อผ่าน MCP, การสร้างแอปที่มี AI เป็นแกน (RAG, Tool Calling, AI Agent), ไปจนถึงการประเมินผล (Evaluation), การวางระบบ Guardrails/Security และการ Deploy สู่ Production พร้อม **Capstone Project** ที่ผู้เรียนสร้างระบบจริงตั้งแต่ต้นจนจบ

- เข้าใจพื้นฐานของ Generative AI, LLM, Transformer, Reasoning Model และความสามารถ/ข้อจำกัดของโมเดลยุค 2026
- เปรียบเทียบและเลือกใช้โมเดลชั้นนำได้อย่างเหมาะสมกับงาน (Claude, GPT, Gemini และโมเดล Open-source)
- ยกระดับจาก **Prompt Engineering** สู่ **Context Engineering** เพื่อสั่งงาน AI ให้ออกแบบ เขียน วิเคราะห์ และแก้ไขโค้ดได้อย่างแม่นยำ
- ใช้งาน **AI Coding Agent** ในกระบวนการพัฒนาจริง ตั้งแต่วางแผน เขียนโค้ด รีแฟกเตอร์ เขียนเทสต์ ไปจนถึงทบทวนโค้ด (Code Review)

- **ประเมิน ทดสอบ และนำระบบ AI ขึ้น Production** ด้วยแนวทาง LLMOps, Evaluation, Observability และ Guardrails
- เข้าใจแนวปฏิบัติที่ดีที่สุด (Best Practices) ด้านความปลอดภัย ความน่าเชื่อถือ และธรรมาภิบาล (Governance) ในการใช้ AI กับงานพัฒนาซอฟต์แวร์จริง

## เนื้อหาการอบรม

### วันที่ 1 — Foundations: Generative AI, LLM & Context Engineering (6 ชั่วโมง)

#### Section 1: Generative AI & LLM Landscape ปี 2026
- แนวคิดของ Generative AI และบทบาทในวงจรการพัฒนาซอฟต์แวร์ (SDLC)
- ภาพรวมตลาดโมเดลปี 2026: Claude (Opus 4.8 / Sonnet 4.6 / Haiku 4.5), GPT-5.5, Gemini 3.5 และโมเดล Open-source
- ความแตกต่างของ AI แบบ Rule-based, LLM-based และ **Reasoning/Agentic Model**
- การเลือกโมเดลให้เหมาะกับงาน: ความสามารถ, ราคา (Cost/Token), ความเร็ว, Context Window และความเป็นส่วนตัวของข้อมูล

#### Section 2: เข้าใจ LLM และ Transformer แบบลึกขึ้น
- โครงสร้างเบื้องหลัง LLM: Transformer, Tokenization และ Context Window
- Reasoning Model และ "Thinking" คืออะไร ต่างจากโมเดลรุ่นก่อนอย่างไร
- RLHF และ Post-training: ทำให้ AI ทำตามคำสั่งและปลอดภัยขึ้นได้อย่างไร
- ทำความเข้าใจ Hallucination และเหตุผลเชิงเทคนิคที่ทำให้ AI ตอบผิด

#### Section 3: จาก Prompt Engineering สู่ Context Engineering สำหรับ Developer
- หลักการเขียน Prompt ที่ดี: Clear, Specific, Role-based, Few-shot
- **Context Engineering**: การจัดการบริบท ไฟล์ความรู้ และ System Prompt ให้ AI เข้าใจโปรเจกต์
- โครงสร้าง Prompt สำหรับงาน Dev: Coding, Debugging, Explain Code, Refactor, Unit Test
- การเขียนไฟล์บริบทมาตรฐาน เช่น `AGENTS.md` และ `CLAUDE.md` เพื่อกำหนดกติกาให้ AI Agent

**Hands-on Labs:**
- ตั้งค่าเครื่องมือ: ChatGPT, Claude, Gemini และ API Key
- ทดลอง Prompt เบื้องต้น: Generate, Refactor, Explain, Translate Code
- เปรียบเทียบผลลัพธ์จากหลายโมเดลในโจทย์เดียวกัน และเขียนไฟล์ `AGENTS.md` ฉบับแรก

### วันที่ 2 — Agentic Coding, MCP & Software Design (6 ชั่วโมง)

#### Section 4: AI Coding Assistant & Agentic Coding
- 3 รูปแบบของเครื่องมือ AI Coding: Inline Suggestion → Chat Assistant → **Autonomous Agent**
- แนะนำเครื่องมือ Agentic Coding ปี 2026: **Claude Code**, **Cursor**, **GitHub Copilot Agent Mode**, **Windsurf**, **OpenAI Codex**
- การใช้ Agent วางแผนและสร้างฟีเจอร์แบบ end-to-end (Plan → Edit → Run → Verify)
- เทคนิคการกำกับ Agent: การให้บริบท, การ Review การเปลี่ยนแปลง (Diff) และการควบคุมขอบเขตงาน

#### Section 5: MCP — เชื่อม AI เข้ากับเครื่องมือและข้อมูลจริง
- **Model Context Protocol (MCP)** คืออะไร และทำไมถูกเรียกว่า "USB-C ของโลก AI"
- สถาปัตยกรรม MCP: Host, Client, Server และการสื่อสารผ่าน JSON-RPC
- การเชื่อม AI เข้ากับ GitHub, ฐานข้อมูล, ระบบไฟล์ และเครื่องมือภายในองค์กรผ่าน MCP Server
- แนวทางความปลอดภัยและสิทธิ์การเข้าถึงเมื่อให้ AI เชื่อมต่อข้อมูลจริง

#### Section 6: AI กับ Software Design & System Architecture
- Generate UML, ERD, Sequence และ Architecture Diagram จาก Prompt (รวมถึง Mermaid/PlantUML)
- ใช้ AI ออกแบบ API ด้วยแนวทาง RESTful / GraphQL และร่าง OpenAPI Spec
- เทคนิค Craft Prompt ให้ AI ช่วยออกแบบโครงสร้างระบบและฐานข้อมูล
- การใช้ AI ประเมินทางเลือกเชิงสถาปัตยกรรม (Trade-off Analysis)

#### Section 7: Legacy Code, Refactoring & AI Code Review
- ให้ AI อธิบายโค้ดเก่า (Explain Code) และทำ Reverse Engineering
- Refactor / Simplify / เพิ่ม Comment และเอกสารให้โค้ดที่ไม่มีเอกสาร
- **AI Code Review**: ใช้ AI ตรวจหา Bug, ช่องโหว่ด้านความปลอดภัย และปัญหาคุณภาพโค้ดก่อน Merge
- Use case: อ่านและปรับปรุงโค้ด Java / Python / TypeScript ที่ไม่มีเอกสาร

**Hands-on Labs:**
- สั่ง AI Agent สร้างระบบ API + Database Schema + Diagram แบบ end-to-end
- ติดตั้งและทดลองใช้ MCP Server (เช่น GitHub / Filesystem) ร่วมกับ AI Agent
- Workshop: ให้ Agent อ่าน Legacy Code, สรุปการทำงาน, รีแฟกเตอร์ และทำ Code Review

### วันที่ 3 — AI Prototyping, Automation, Testing & Best Practices (6 ชั่วโมง)

#### Section 8: AI Prototyping & UI Generation
- ใช้ AI สร้าง UI ด้วย HTML, Tailwind CSS และ Component สำเร็จรูป
- เครื่องมือสร้างเว็บ/UI จาก Prompt ปี 2026: v0, Lovable, Bolt, Framer AI และอื่น ๆ
- Generate Static/Full-stack Web และ Deploy บน GitHub Pages / Vercel / Netlify
- การปรับแต่งและต่อยอดโค้ดที่ AI สร้างให้ใช้งานจริง

#### Section 9: AI-powered Chatbot, RAG & Assistant (เบื้องต้น)
- แนวคิด **RAG (Retrieval-Augmented Generation)** และเหตุผลที่จำเป็นสำหรับ Chatbot องค์กร
- สร้าง Chatbot ตอบคำถามจากเอกสาร/ฐานความรู้ภายใน
- ใช้ Claude / ChatGPT สร้าง FAQ Bot และ Coding Assistant
- ภาพรวมการต่อยอดด้วย Vector Database และ Embedding (ปูพื้นสู่วันที่ 4)

#### Section 10: AI Testing, Automation & Spec-driven Development
- ใช้ AI เขียน Unit Test, Integration Test และสร้างข้อมูลทดสอบ
- **Spec-driven Development**: เขียนสเปก/ข้อกำหนดให้ชัดเจน แล้วให้ AI สร้างและตรวจสอบงานตามสเปก
- การเชื่อม AI เข้ากับ CI/CD และระบบ Automation
- การวัดคุณภาพและความถูกต้องของงานที่ AI สร้าง

#### Section 11: Best Practices, Security & Governance (สำหรับนักพัฒนา)
- วิธีเชื่อม AI เข้า Dev Lifecycle: Plan → Build → Test → Review → Deploy
- ข้อควรระวัง: Security, Hallucination, Data Privacy, Copyright และ License ของโค้ด
- ธรรมาภิบาลการใช้ AI ในองค์กร (AI Governance) และความรับผิดชอบของนักพัฒนา
- การสร้าง Prompt/Context Library ส่วนตัวและของทีมเพื่อใช้ซ้ำ

**Hands-on Labs:**
- สร้าง Web App หรือ UI ด้วย Prompt และนำขึ้น Hosting จริง
- ทดลองสร้าง Chatbot แบบ RAG เบื้องต้นตามโจทย์ Workshop
- สรุป Prompt/Context ที่ใช้ได้จริง รวมเป็น Prompt Library ส่วนตัว

### วันที่ 4 — AI Engineering: Building AI-Powered Applications (6 ชั่วโมง)

> **เป้าหมายของวัน:** ก้าวจาก "ผู้ใช้ AI ช่วยพัฒนา" สู่ "ผู้สร้างแอปพลิเคชันที่มี AI เป็นแกนหลัก" ผ่านการเขียนโปรแกรมเรียกใช้ LLM, การทำ RAG ระดับ Production, การให้ AI เรียกใช้เครื่องมือ และการประกอบ AI Agent ของตนเอง

#### Section 12: LLM API & Application Integration
- พื้นฐานการเรียกใช้ LLM ผ่าน API: Anthropic, OpenAI และ Gemini
- พารามิเตอร์สำคัญ: Temperature, Max Tokens, System Prompt, Structured Output (JSON Mode)
- การออกแบบ Prompt Template และจัดการ Context Window อย่างมีประสิทธิภาพ
- Streaming, Error Handling, Rate Limit และการจัดการต้นทุน (Token Cost) ในแอปจริง

#### Section 13: RAG ระดับ Production (Retrieval-Augmented Generation)
- สถาปัตยกรรม RAG แบบเต็ม: Ingest → Chunk → Embed → Store → Retrieve → Generate
- **Embeddings** และ **Vector Database** (เช่น pgvector, Qdrant, Pinecone, Chroma)
- เทคนิคปรับคุณภาพการค้นคืน: Chunking Strategy, Hybrid Search, Re-ranking
- การลด Hallucination ด้วยการอ้างอิงแหล่งข้อมูล (Citation/Grounding)

#### Section 14: Tool/Function Calling & การสร้าง AI Agent
- **Tool / Function Calling**: ให้ LLM เรียกใช้ฟังก์ชัน, API และฐานข้อมูลได้
- แนวคิด AI Agent: การวางแผน (Planning), การใช้เครื่องมือ (Tool Use) และการวนซ้ำ (Loop)
- รูปแบบสถาปัตยกรรม Agent: Single-agent, Multi-agent และ Workflow Orchestration
- ภาพรวม Framework สำหรับสร้าง Agent (เช่น LangChain, LlamaIndex, LangGraph)

#### Section 15: สร้าง MCP Server ของตัวเอง
- ทบทวนสถาปัตยกรรม MCP และยกระดับสู่การ "เป็นผู้สร้าง" แทนผู้ใช้
- การพัฒนา MCP Server เพื่อเปิดเครื่องมือ/ข้อมูลขององค์กรให้ AI Agent เรียกใช้
- การออกแบบ Tools, Resources และ Prompts ภายใน MCP Server
- แนวทางความปลอดภัยและการจัดการสิทธิ์ (Authorization) ของ MCP Server

**Hands-on Labs:**
- เขียนโปรแกรมเรียกใช้ LLM API พร้อม Structured Output
- สร้างระบบ RAG ขนาดเล็กที่ตอบคำถามจากเอกสารของผู้เรียนเอง
- สร้าง AI Agent ที่เรียกใช้เครื่องมือ (เช่น ค้นข้อมูล / คำนวณ / เรียก API)
- พัฒนา MCP Server อย่างง่ายและเชื่อมกับ AI Agent

### วันที่ 5 — Production, LLMOps, Evaluation & Capstone (6 ชั่วโมง)

> **เป้าหมายของวัน:** นำระบบ AI ที่สร้างขึ้นไปใช้งานจริงอย่างน่าเชื่อถือ ปลอดภัย และคุ้มค่า ครอบคลุมการประเมินผล การเฝ้าระวัง การวางระบบป้องกัน และการ Deploy พร้อมปิดท้ายด้วย Capstone Project

#### Section 16: Evaluation & Testing ของระบบ AI
- ทำไมระบบ AI จึงทดสอบยากกว่าโปรแกรมทั่วไป (Non-deterministic)
- การออกแบบชุดประเมินผล (Eval Set) และเมตริกสำหรับ LLM/RAG
- **LLM-as-a-Judge** และการประเมินอัตโนมัติ
- การทำ Regression Testing เมื่อเปลี่ยนโมเดลหรือ Prompt

#### Section 17: LLMOps & Observability
- วงจร LLMOps: Prompt/Model Versioning, Deployment และการ Monitor
- การติดตามต้นทุน (Cost), ความเร็ว (Latency) และคุณภาพคำตอบใน Production
- Logging, Tracing และ Observability สำหรับแอปที่ขับเคลื่อนด้วย LLM
- เทคนิคลดต้นทุนและเพิ่มประสิทธิภาพ: Caching, Routing, Model Selection

#### Section 18: AI Security, Guardrails & Responsible AI
- ภัยคุกคามเฉพาะของ LLM: **Prompt Injection**, Jailbreak, Data Leakage (อ้างอิง OWASP Top 10 for LLM)
- การวาง **Guardrails**: Input/Output Validation, Content Filtering และ Policy
- ความเป็นส่วนตัวของข้อมูลและการจัดการข้อมูลอ่อนไหว (PII)
- หลัก Responsible AI: Bias, Transparency และความรับผิดชอบในการนำไปใช้

#### Section 19: Deployment & Capstone Project
- แนวทาง Deploy ระบบ AI: Serverless, Container และ Managed Platform
- การออกแบบสถาปัตยกรรมแอป AI ให้พร้อมขยาย (Scalability) และทนทาน (Reliability)
- การส่งมอบและดูแลรักษาระบบ AI หลัง Go-live
- สรุปบทเรียนและแนวทางพัฒนาตนเองสู่สาย AI Engineer

**Hands-on Labs & Capstone:**
- สร้าง Eval Set และประเมินคุณภาพระบบ RAG/Agent ที่ทำในวันที่ 4
- ทดลองโจมตีแบบ Prompt Injection และวาง Guardrails ป้องกัน
- **Capstone Project:** สร้างแอปพลิเคชันที่ขับเคลื่อนด้วย AI แบบ end-to-end (ออกแบบ → สร้าง RAG/Agent → ประเมินผล → วาง Guardrails → Deploy) และนำเสนอผลงาน
