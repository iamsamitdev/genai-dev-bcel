### 1.1 Claude Code คืออะไร

Claude Code คือเครื่องมือที่ให้เรา "สั่ง AI ให้สร้างซอฟต์แวร์" ด้วยภาษาคน — Claude จะอ่านไฟล์ในโปรเจกต์ วางแผน เขียนโค้ด แก้ไข ทดสอบ และปรับปรุงให้ โดยที่เราไม่ต้องพิมพ์โค้ดเอง ต่างจากการคุยกับ AI ทั่วไปตรงที่มันลงมือทำกับไฟล์จริงในโปรเจกต์ได้

```
คุยกับ AI ทั่วไป                     Claude Code (Agentic Coding)
┌──────────────────────────┐        ┌──────────────────────────────┐
│ ให้โค้ดมาเป็นข้อความ      │        │ สร้าง/แก้ไฟล์ในโปรเจกต์จริง   │
│ เรา Copy ไปวางเอง        │   ⟶   │ รันและทดสอบให้                │
│ แก้เองเมื่อ Error         │        │ เห็น Error แล้วแก้ต่อเอง       │
│ ไม่เห็นภาพรวมโปรเจกต์     │        │ อ่านทั้งโปรเจกต์ก่อนลงมือ      │
└──────────────────────────┘        └──────────────────────────────┘
```

### 1.2 Agentic Coding — AI ทำงานเป็นขั้นตอนอย่างไร

วงจรการทำงานเหมือน Agent Loop ใน Day 1 แต่บริบทเป็นงานพัฒนา:

```
โจทย์ (Brief)
   ↓
[อ่าน] โครงสร้างโปรเจกต์ + ไฟล์ที่เกี่ยวข้อง
   ↓
[วางแผน] จะสร้าง/แก้ไฟล์อะไรบ้าง
   ↓
[ลงมือ] เขียนโค้ด สร้างหน้าเว็บ
   ↓
[ทดสอบ] เปิดดูผล หา Error
   ↓
[ปรับปรุง] แก้ตาม Feedback ของเรา
```

### 1.3 สิ่งที่คนไม่เขียนโค้ดควรรู้

ไม่ต้องเข้าใจไวยากรณ์โปรแกรม แต่ควรรู้ "คำศัพท์โครงสร้าง" พอคุยกับ AI รู้เรื่อง:

| คำ               | ความหมายแบบง่าย                                       |
| ---------------- | ----------------------------------------------------- |
| โฟลเดอร์โปรเจกต์ | กล่องเก็บไฟล์ทั้งหมดของงานหนึ่งชิ้น                   |
| ไฟล์ HTML        | หน้าเว็บ 1 หน้า (โครงสร้างหน้า)                       |
| CSS              | หน้าตา/สไตล์ (สี ฟอนต์ ระยะห่าง)                      |
| ข้อมูล (data)    | ไฟล์ตัวเลข/ข้อความที่เอามาแสดงผล เช่น `.csv`, `.json` |
| รัน (run)        | สั่งให้โปรแกรมทำงานเพื่อดูผล                          |
| ทดสอบ (test)     | ตรวจว่าได้ผลตามที่ต้องการไหม                          |

### 1.4 ข้อควรระวังในการใช้ AI Coding

> ⚠️ **อย่าเชื่อผลลัพธ์ทันที** — AI สร้างของได้เร็ว แต่อาจมีจุดผิด ต้องตรวจและ Feedback เป็น
>
> - ตรวจหน้าตา/เนื้อหาด้วยตาทุกครั้งก่อนนำไปใช้
> - ถ้าผลไม่ตรงใจ ให้บอกเฉพาะจุดที่อยากแก้ ไม่ต้องเริ่มใหม่ทั้งหมด
> - ข้อมูลสำคัญ (ตัวเลข ราคา) ตรวจให้ตรงกับต้นฉบับเสมอ

### 1.5 Vibe Coding อย่างมีวินัย - สั่ง AI ด้วยหลัก 5C

การ "คุย" ให้ AI สร้างงานให้เราด้วยภาษาคน เรียกว่า **Vibe Coding** ข้อดีคือเร็วและไม่ต้องเขียนโค้ด แต่ถ้าสั่งแบบไม่มีวินัย มักเจอ 3 ปัญหานี้:

- AI สร้างของออกมา แต่ไม่ตรงกับที่เราต้องการจริง ๆ
- คุยไปเรื่อย ๆ จน AI ลืมบริบทเดิม ทำงานซ้ำหรือขัดกับของเก่า
- ได้งานที่ "พอใช้ได้" แต่ไม่ตรงมาตรฐานหรือแบรนด์ของเรา

ทางแก้คือสั่งงานให้ครบตามหลัก **5C**:

| หลัก                               | ความหมายแบบง่าย                    | ตัวอย่างสำหรับงานเว็บ                                |
| ---------------------------------- | ---------------------------------- | ---------------------------------------------------- |
| **Context** (บริบท)                | บอกว่างานนี้คืออะไร เพื่อใคร       | "ทำ Landing Page ขายคอร์ส Excel สำหรับพนักงานออฟฟิศ" |
| **Constraint** (กรอบ)              | ข้อจำกัดหรือกฎที่ต้องทำตาม         | "ใช้สีแบรนด์น้ำเงิน #003087 และต้องรองรับมือถือ"     |
| **Concrete** (รูปธรรม)             | ระบุผลลัพธ์ให้ชัด จับต้องได้       | "เป็นไฟล์ HTML หน้าเดียว เปิดในเบราว์เซอร์ได้เลย"    |
| **Context File** (อ้างอิงของที่มี) | ชี้ไปที่ไฟล์หรือข้อมูลที่เตรียมไว้ | "ใช้เนื้อหาจากไฟล์ Brief และรีวิวที่แนบให้"          |
| **Check** (ขอให้ตรวจก่อน)          | ให้ AI วางแผนหรือยืนยันก่อนลงมือ   | "วางโครงหน้าเว็บให้ดูก่อน แล้วค่อยลงมือสร้าง"        |

**เทียบให้เห็นภาพ:**

```
❌ สั่งลอย ๆ:   ทำหน้าขายคอร์สให้หน่อย

✅ สั่งมีวินัย:  ทำ Landing Page ขายคอร์ส Excel 1 หน้า สำหรับพนักงานออฟฟิศ
              ใช้สีน้ำเงิน #003087 ดูมืออาชีพ รองรับมือถือ
              ใช้เนื้อหาจากไฟล์ Brief ที่แนบ วางโครงให้ดูก่อนแล้วค่อยสร้าง
```

> 💡 **จำง่าย ๆ:** ยิ่งสั่งครบ 5C ผลลัพธ์ยิ่งตรงและแก้น้อยรอบ หลัก 5C นี้คือหัวใจของการเขียน Brief ที่ดี ซึ่งเราจะลงลึกใน Module 2 ถัดไป

### 2.1 ทำไม Brief สำคัญกว่าโค้ด

สำหรับคนไม่เขียนโค้ด ทักษะที่สำคัญที่สุดคือ **การเขียน Brief** — บอก AI ให้ครบว่าต้องการอะไร ยิ่ง Brief ชัด ผลลัพธ์ยิ่งตรง และแก้น้อยรอบ

### 2.2 โครงสร้าง Brief ที่ดี

| หัวข้อ              | คำถามที่ต้องตอบ                                 |
| ------------------- | ----------------------------------------------- |
| กลุ่มเป้าหมาย       | ใครคือคนที่จะเข้าเว็บนี้                        |
| จุดขาย              | ทำไมเขาต้องสนใจ/ซื้อ                            |
| โครงหน้าเว็บ        | มี Section อะไรบ้าง เรียงอย่างไร                |
| สีและสไตล์          | โทนสี อารมณ์ ความรู้สึก                         |
| Call to Action      | อยากให้ผู้เข้าชมทำอะไร (โทร/สั่งซื้อ/ลงทะเบียน) |
| ข้อมูลสินค้า/บริการ | รายละเอียด ราคา จุดเด่น                         |

> 📎 มีเทมเพลตและตัวอย่างพร้อมใช้: `01_brief_template.md`, `01_good_brief_example.md`, `01_bad_brief_examples.md` และ brand guideline ตัวอย่าง `01_brand_guidelines_sample.md`

### 2.3 ตัวอย่าง Brief ที่ครบถ้วน

```
สร้าง Landing Page ขายคอร์สออนไลน์ "Excel for Business" 1 หน้า

กลุ่มเป้าหมาย: พนักงานออฟฟิศที่อยากใช้ Excel เก่งขึ้น
จุดขาย: เรียนจบทำงานจริงได้ทันที มีเทมเพลตให้ใช้ฟรี
โครงหน้าเว็บ (เรียงตามนี้):
  1. Hero: ชื่อคอร์ส + ประโยคดึงดูด + ปุ่ม "สมัครเลย"
  2. ปัญหาที่ผู้เรียนเจอ (3 ข้อ)
  3. สิ่งที่จะได้เรียน (6 หัวข้อ)
  4. ราคา + โปรโมชัน
  5. รีวิวผู้เรียน (3 อัน)
  6. FAQ (5 ข้อ)
  7. ช่องทางติดต่อ
สี: น้ำเงิน #003087 + ขาว ดูเป็นมืออาชีพ
CTA หลัก: ปุ่ม "สมัครเลย" ราคา 1,990 บาท
ทำให้รองรับมือถือด้วย
```

### 2.4 ให้ Claude วางแผนก่อนลงมือ

> 💡 **เทคนิคสำคัญ:** ก่อนให้สร้างจริง สั่งว่า _"ช่วยวางแผนโครงสร้างหน้าเว็บและถามกลับถ้ามีอะไรไม่ชัด ก่อนเริ่มเขียน"_ — Claude จะสรุปแผนให้เราอนุมัติก่อน ลดงานแก้ทีหลัง

### 2.5 ตรวจงานเว็บเบื้องต้นแบบคนไม่เขียนโค้ด

เช็กลิสต์ดูด้วยตา: เนื้อหาครบทุก Section ไหม, ปุ่ม CTA ชัดและกดได้ไหม, ข้อความสะกดถูกไหม, ดูบนมือถือแล้วล้นจอไหม, สีและภาพอ่านง่ายไหม

### 3.2 Section สำคัญที่ทำให้เว็บน่าเชื่อถือ

แนะนำให้เว็บขายของมีครบ: **ปัญหาลูกค้า → สิ่งที่ได้รับ → รีวิว → FAQ → ช่องทางติดต่อ** เพราะช่วยให้คนตัดสินใจง่ายขึ้น

### 3.3 ปรับแต่งเป็นรอบ (Iterative Refinement)

หลังได้เว็บฉบับร่าง สั่งแก้เฉพาะจุด ไม่ต้องเริ่มใหม่:

```

- เปลี่ยนปุ่ม CTA ให้เด่นขึ้น สีส้ม ตัวใหญ่ขึ้น
- เพิ่มรูปประกอบใน Hero
- ข้อความหัวข้อยาวไป ช่วยตัดให้กระชับ
- เพิ่มส่วน "รับประกันความพอใจ" ก่อน FAQ
```

### 5.1 AI Agent คืออะไรในภาษาคนทำงาน

AI Agent คือ "ผู้ช่วยเฉพาะทาง" ที่เราออกแบบให้ทำงานหนึ่งอย่างได้ดีและทำซ้ำได้ — ต่างจาก Chatbot ธรรมดาตรงที่มีเป้าหมายชัด รับข้อมูลเข้า ประมวลผลตามขั้นตอนที่เรากำหนด แล้วให้ผลลัพธ์ที่คาดเดาได้

| Chatbot ธรรมดา     | AI Agent                  |
| ------------------ | ------------------------- |
| ตอบคำถามทั่วไป     | ทำงานเฉพาะที่ออกแบบไว้    |
| ไม่มีขั้นตอนตายตัว | มี Workflow ชัดเจน        |
| ผลลัพธ์หลากหลาย    | ผลลัพธ์เป็นรูปแบบเดียวกัน |

### 5.2 ตัวอย่าง Agent ที่ทำได้ในคลาส

- Agent สรุปข้อมูลจากไฟล์ (โยนไฟล์ให้ → ได้สรุปตามรูปแบบที่กำหนด)
- Agent ตอบคำถามจากข้อมูลธุรกิจ (ถามเรื่องสินค้า/ราคา → ตอบจากไฟล์ข้อมูล)
- Agent ช่วยร่างคอนเทนต์ (ให้หัวข้อ → ได้โพสต์ตามโทนแบรนด์)
- Agent จัดหมวดหมู่ลูกค้า (ให้รายชื่อ → จัดกลุ่มตามเกณฑ์)
- Agent สร้าง Checklist งานซ้ำ

### 5.3 วิธีออกแบบ Agent (5 ช่อง)

```
Agent Name: ___________
1. เป้าหมาย — ช่วยทำอะไร ให้ผลลัพธ์แบบไหน
2. ข้อมูลเข้า (Input) — รับอะไร (ไฟล์/ข้อความ/คำถาม)
3. ขั้นตอนคิด (Process) — ทำอะไรทีละขั้น
4. ผลลัพธ์ (Output) — รูปแบบ/ไฟล์ที่ได้
5. ข้อจำกัด — ห้ามทำอะไร / ต้องให้คนตรวจตรงไหน
```

### 5.4 ตัวอย่างการสั่งสร้าง Agent

```
สร้าง AI Agent ชื่อ "ผู้ช่วยตอบลูกค้า" สำหรับร้านของฉัน
เป้าหมาย: ตอบคำถามลูกค้าจากไฟล์ข้อมูลสินค้า product_info.md
ข้อมูลเข้า: คำถามลูกค้า (เช่น ราคา การจัดส่ง การรับประกัน)
ขั้นตอน: อ่านไฟล์ → หาคำตอบที่ตรง → ตอบสุภาพเป็นภาษาไทย
ผลลัพธ์: คำตอบสั้นกระชับ + ถ้าไม่มีข้อมูลให้บอกว่า "จะตรวจสอบและแจ้งกลับ"
ข้อจำกัด: ห้ามมั่วราคา ถ้าไม่มีในไฟล์ และห้ามสัญญาส่วนลดเอง
```

### 5.5 ทดสอบ Agent ให้ตอบตรงและไม่มั่ว

> ⚠️ **ทดสอบก่อนใช้จริงเสมอ:** ใช้ชุดคำถาม `04_agent_test_questions.md` ที่แบ่งเป็น 3 กลุ่ม — (A) มีคำตอบในไฟล์ ต้องตอบถูก, (B) ไม่มีคำตอบ ต้องยอมรับว่าไม่รู้ (ไม่แต่งเรื่อง), (C) เกินขอบเขต ต้องส่งต่อทีมงาน

### 6.2 Slash Command — คำสั่งลัดสำหรับงานซ้ำ

Slash Command คือคำสั่งสั้น ๆ (ขึ้นต้นด้วย `/`) ที่ผูกกับชุดงานที่เราทำบ่อย เช่น:

| -------------- | ----------------------------------------- |
| `/new-landing` | สร้างโครง Landing Page ตามสไตล์แบรนด์เรา  |
| `/add-section` | เพิ่ม Section มาตรฐาน (รีวิว/FAQ/ติดต่อ)  |
| `/review-web`  | ตรวจเว็บตาม Checklist (มือถือ, CTA, สะกด) |

### 6.3 สร้าง Skill ของตัวเองให้ทำงานตามมาตรฐานเรา

ใช้กรอบ 6 ด้านเดียวกับ Day 1 ("ใคร–เมื่อไร–อะไรเข้า–อะไรออก–ทำยังไง–เช็คยังไง") เช่น:

```
สร้าง Skill ชื่อ landing-page-builder
- ใช้เมื่อ: "สร้าง landing page", "ทำหน้าขายสินค้า"
- Input: ชื่อสินค้า, จุดขาย, ราคา, สีแบรนด์
- Output: ไฟล์ HTML หน้าเดียว responsive มีครบ Hero/ปัญหา/สิ่งที่ได้/รีวิว/FAQ/ติดต่อ
- เทคนิค: ใช้โครง HTML+CSS ในไฟล์เดียว, ปุ่ม CTA เด่น, ตรวจ contrast
- Verify: เปิดดูทุก Section ครบ, ทดสอบมือถือ, สะกดถูก
```

### 6.4 ไฟล์ CLAUDE.md - สมองของโปรเจกต์

นอกจาก Skill แล้ว ยังมีไฟล์พิเศษชื่อ **CLAUDE.md** ที่ Claude Code จะอ่าน **อัตโนมัติทุกครั้ง** ที่เราเปิดโปรเจกต์ ทำหน้าที่เหมือน "สมอง" หรือความจำถาวรของงานชิ้นนั้น เก็บมาตรฐานที่เราไม่อยากพิมพ์ซ้ำทุกครั้ง เช่น สีแบรนด์ โทนภาษา หรือสิ่งที่ห้ามทำ

**CLAUDE.md ต่างจาก Skill อย่างไร:**

|            | CLAUDE.md                 | Skill                            |
| ---------- | ------------------------- | -------------------------------- |
| เปรียบเป็น | ความจำ/มาตรฐานของโปรเจกต์ | ความสามารถ/ขั้นตอนทำงาน          |
| ทำงานเมื่อ | อ่านเองทุกครั้งที่เปิดงาน | เรียกใช้เมื่อเราสั่ง             |
| เก็บอะไร   | กฎ แบรนด์ สิ่งที่ห้ามทำ   | วิธีทำงานอย่างหนึ่งให้ได้มาตรฐาน |

**ตัวอย่าง CLAUDE.md สำหรับงานเว็บธุรกิจ (เขียนเป็นภาษาคนล้วน ไม่ต้องเขียนโค้ด):**

```
# เว็บร้านกาแฟ Aroma

## แบรนด์ของเรา
- โทนสี: น้ำตาลอบอุ่น + ครีม
- อารมณ์: อบอุ่น เป็นกันเอง สะอาดทันสมัย

## มาตรฐานทุกหน้าเว็บ
- ต้องรองรับมือถือเสมอ
- ปุ่มสั่งซื้อต้องเด่นและกดง่าย
- ใช้ภาษาสุภาพ เป็นกันเอง

## สิ่งที่ห้ามทำ
- ห้ามใส่ราคาที่ไม่ได้ยืนยัน
- ห้ามใช้สีนอกโทนแบรนด์
```

พอมีไฟล์นี้ ครั้งต่อไปแค่สั่งว่า "ทำหน้าโปรโมชันใหม่" Claude ก็จะใช้สีแบรนด์ โทนภาษา และมาตรฐานเดิมให้อัตโนมัติ โดยเราไม่ต้องอธิบายซ้ำ

> 💡 **สรุป:** เขียนครั้งเดียว ใช้ได้ทุกครั้งที่เปิดงาน และแชร์ให้ทีมใช้มาตรฐานเดียวกันได้ - CLAUDE.md ยิ่งชัด AI ยิ่งทำงานตรงใจโดยไม่ต้องบอกซ้ำ
